STATUS UPDATE FROM CLOVER SESSION (integration ready):
- All sync/device/order routes now accept an optional `x-merchant-id` HTTP header as a fallback to the cookie.
- This lets the web app call the Clover backend even if running cross-origin or not sharing cookies.
- Example web app calls:

  const CLOVER_BACKEND = "http://localhost:3000";
  const MERCHANT_ID = "localStorage.getItem('clover_merchant_id')";

  // Pull menu from Clover
  await fetch(`${CLOVER_BACKEND}/api/sync/pull`, {
    method: "POST",
    headers: { "x-merchant-id": MERCHANT_ID },
  });

  // Push menu to Clover
  await fetch(`${CLOVER_BACKEND}/api/sync/push`, {
    method: "POST",
    headers: { "x-merchant-id": MERCHANT_ID },
  });

  // Scan devices
  await fetch(`${CLOVER_BACKEND}/api/devices/scan`, {
    method: "POST",
    headers: { "x-merchant-id": MERCHANT_ID },
  });

  // List devices
  await fetch(`${CLOVER_BACKEND}/api/devices`, {
    headers: { "x-merchant-id": MERCHANT_ID },
  });

  // Push order to Clover POS (pay in person)
  await fetch(`${CLOVER_BACKEND}/api/orders/push`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      cloverMerchantId: MERCHANT_ID,
      lineItems: [
        { name: "Burger", price: 1099, quantity: 1, item: { id: "clover_item_id" } },
      ],
      note: "Pickup order #123",
      orderType: null,
    }),
  });

- After successful OAuth, the web app should save merchant_id (from the cookie or callback URL) into its own localStorage/supabase session.
